In creating the prototypes, I/we implemented the following features (at least 2 for an individual, 3 for a pair)

  1. Logging in with user name and password
  2. Ability to add a new beer note (XML file) entry
(describe briefly each feature including the technique(s) used to implement this feature (one list item per feature))
  1. When the user logs in, the PHP function checks the entered user name and password for correctness. The password uses MD5 hashing. If they are correct, the login form is replaced with a "box" which shows a link to their user page and also a "Sign Out" button. Also, the link to the user page appears in the top navigational bar. If the user enters wrong credentials, the input fields turn red and a message "Wrong User Name or Password" appears. The colour change is done using JavaScript function.

    You can log in using following credentials:
    1. User: beer_lover Password: beer
    2. User: Boris Password: beer1
  2. The user fills a beer tasting form and based on that, a new XML file is created and presented in the beer notes listing. The form is created using HTML and XML file using PHP SimpleXML functions. The beer note can also be set private which means that it will appear in the users beer note list but not in the general list (unless they are beer notes by the user currently logged in). User can also delete their beer notes by clicking the cross next to a list item in their page. I have set up private beer notes for both, Boris and beer_lover.
